## 1. 核心開發精神 (Core Principles)

1. **欄位為王 (Field-First)**：業務資料的欄位與邏輯必須優先對齊 Prototype，這是功能的靈魂，UI 排版讓位給它。
2. **平台一致性 (Platform-Consistency)**：優先使用 `vue-pure-admin` + Element Plus 原生組件，**嚴禁自行開發 CSS 框架**，除非平台完全不支援（如 `WChoiceUploader`）。
3. **物理隔離**：所有自定義代碼位於 `src/views-custom/`，嚴禁修改 `src/views/` 官方目錄。
4. **最小 CSS 原則**：儘量使用 `var(--el-*)` 設計 Token 與 Tailwind 原子化 class，自定義 `<style scoped>` 應保持 15 行以內。

## 3. 當前已知 Bug 與待辦事項 (Pending Tasks)

### 🔴 高優先級：功能缺失

#### BUG-01: 金額欄位消失 + 結算總計無連動

- **現象**：勾選服務項目後，Sticky Footer 的 `報價合計` 不變動（仍顯示 `$0`）
- **懷疑原因**：
  - `create.vue` 直接調用 `useQuotationForm()`，但 `calculateTotals` 依賴 `watch(deep)` 對 `form.regions` 的監測，需確認 reactive 響應鏈是否斷裂
  - `item.selectedOptionIndex` 觸發時，應回更新 `item.price` 與 `item.time_min`，但目前 `form-hook.ts` 的 watch 回調是否有在 `reactive` 上正確觸發需要重新驗證
- **診斷指令**：在 `calculateTotals` 中加入 `console.log`，確認 watch 是否被觸發
- **修復建議**：考慮使用 `computed` 取代 `watch` 計算 `totalPrice`，更符合 Vue 3 響應式語意

## 4. 關鍵設計決策紀錄 (Decisions Log)

| 決策           | 選擇                             | 原因                                    |
| -------------- | -------------------------------- | --------------------------------------- |
| 結算資訊佈局   | `sticky footer`（非右側欄）      | 右側欄會壓縮項目寬度，導致展開後跑版    |
| Footer 定位    | `position: sticky`（非 `fixed`） | `fixed` 會忽略 sidebar 寬度，蓋住選單   |
| 展開動畫       | `<el-collapse-transition>`       | 框架內建，避免自訂 CSS keyframe         |
| 上傳組件       | 自研 `WChoiceUploader`           | Element Plus el-upload 無法支援封面標記 |
| 服務項目數量欄 | `:md="4"` + `w-[90px]`           | 數量不超過 3 位數，不需要寬欄           |
| 條件屬性欄     | `:md="5"`                        | 縮減一半釋放空間給照片欄                |
