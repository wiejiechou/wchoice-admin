# WChoice Admin 開發規範與 AI 協作契約

> 本文件定義了專案在技術實作、UI 介面對齊與 AI Agent 交付時的強制性規範，旨在降低開發錯誤率並提升程式碼一致性。
> **最後更新日期**：2026-04-20 | **版本**：v1.8

---

## 🏗️ 技術棧與前端規範

### 0. 黃金範本 (Golden Sample) - 強制參考

- **首選參考 (標準維護)**：`src/views-custom/internal/customer/` [NEW]
  - 特徵：600px 抽屜、標準 CRUD、yyyy-MM-dd 日期格式、純淨原生風格。
- **次要參考 (複雜業務)**：`src/views-custom/quotation/`
  - 特徵：100% 全螢幕、多組件累加邏輯。

### 1. 代碼風格與格式規範 (Styling & Formatting)

- **多屬性標籤排版**：
  - 當一個標籤/組件擁有 **3 個屬性以上** 或 **單行長度過長** 時，必須將屬性進行換行排版，每個屬性獨立佔據一行。
- **排版防禦 (三向原則)**：
  - 複雜邏輯（Filter 內判斷 > 1）-> 強制區塊換行與 `return`。
  - 單行長度 > 80 字 -> 主動折行。

### 2. Tailwind CSS v4 強制規範

- **版本對齊**：嚴格遵循 Tailwind CSS v4 規範。
- **禁止任意值 (Anti-Arbitrary Values)**：全面禁止使用 `w-[150px]`、`mt-[12px]` 等任意值語法。
- **強制使用 Tokens**：必須選取系統 Design Tokens 中最接近的標籤（例如：`w-45!`、`max-w-3xl`）。
- **後綴驚嘆號**：採用後置 `!` 語法（例如：`w-45!`、`pt-3!`），嚴禁使用 v3 的前置 `!`。
- **樣式隔離**：在 Vue 組件中使用 `<style lang="scss" scoped>`，並透過 `:deep()` 穿透。

### 3. 組件開發策略 (Reusability & Consultation)

- **元件重用優先**：優先搜尋現有組件（如 `PureTableBar`、`pure-table`）。
- **原生組件嚴格化**：優先使用 **Element Plus 原生組件** 開發，避免過度自定義 CSS。
- **開發前諮詢制度 [NEW]**：
  - **新組件諮詢**：如欲建立的功能與現有組件相似，**必須先諮詢用戶** 確認是否有必要建立，嚴禁重複造輪子。
  - **寬度諮詢**：所有抽屜 (Drawer) 開發前，必須根據欄位量評估推薦寬度（如 `600px`, `800px`, `100%`），並 **主動與用戶對齊後** 才動工。

## 🎨 UI 互動規範

### 1. 複雜表單與 RWD 支援

- **規模評估制**：不再強制全螢幕。依據欄位量與佈局複雜度，開發前提報抽屜寬度方案。
- **RWD 閘口 [NEW]**：針對複雜表單的 RWD 支援，若實作上存在多重方案或效能疑慮，**必須先提出方案進行諮詢**，經用戶同意後才動工。

### 2. 數據格式化標準 (Display Standard) [NEW]

- **日期顯示**：畫面涉及到日期顯示的部分，一律採用 **`yyyy-MM-dd`** 格式 (資料庫為 `datetime` 資料類型)。
- **一致性**：確保列表、表單與 Excel 匯出之日期呈現風格統一。

### 3. 狀態連動邏輯

- **宣告式計算 (Computed-over-Watch)**：衍生數值強制使用 `computed`。嚴禁使用深度 `watch` 手動改寫 state。

### 4. 複雜組件狀態同步 (Local Accumulator Pattern)

- **問題背景**：批次上傳等非同步事件觸發時，`props.modelValue` 未必即時更新。
- **解決方案**：
  1. 引入 **`localFiles` (ref)** 作為元件內部的「單一事實來源」。
  2. 同步操作本地 ref，確保連續事件正確累加。
  3. `watch(props.modelValue)` 處理編輯回填。

## 🛠️ 開發流程 SOP

### 1. 快速特徵對齊流程 (Copy & Refactor)

1. **物理複製 (Copy)**：複製相似功能的目錄。
2. **領域映射 (Domain Mapping)**：全面替換主題關鍵字與 TS 欄位定義。
3. **架構清理 (Refactoring)**：移除舊功能殘留邏輯。
4. **規範檢查**：執行 Tailwind v4 與日期格式化檢查。

## 🛡️ AI Agent 交付檢查清單 (CI/CD Checklist)

### 1. 交付前必讀
- [ ] **Tailwind v4 Token 檢測**：全面消除 `w-[...]`，強制使用系統標籤。
- [ ] **抽屜寬度確認**：是否已與用戶對齊過本次 Drawer 寬度？
- [ ] **日期格式檢查**：是否符合 `yyyy-MM-dd`？
- [ ] **原生組件核實**：是否優先使用了 Element Plus 原生組件？
- [ ] **清理未使用引入**：移除 unused imports 防止 husky 攔截。
